Structuur

Voor het komende project, de meesterproef, wil ik graag mezelf beter voorbereiden. Ik wil vanaf het begin een gedetaileerde wiki bijhouden en een duidelijk set-up maken voor mijn code. Nu merk ik vaak dat ik te slordig kan werken waardoor ik soms in de problemen kom. Ik hoop dat ik door een gestructureerd plan op te stellen, beter overzicht heb en dus minder problemen krijg verder in het project. Een manier om dit te doen is het goed bijhouden van mijn proces in de vorm van een wiki, maar ook door breakdown schetsen te maken. Hiermee kan ik gefocust werken aan de code in plaats van aan de vormgeving. Daarnaast wil ik meer aandacht besteden aan een gestructureerde en duidelijke code. Ik werk te vaak slordig, waardoor ik snel mijn eigen code niet meer kan ontcijferen. Ik kan best netjes werken, ik moet het alleen echt gaan doen.
